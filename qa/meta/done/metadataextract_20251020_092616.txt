# SAP SuccessFactors Metadata Extraction Report
## Queensland Department of Education - Organizational Design System

### Executive Summary
This metadata extraction focuses on the organizational design requirements for the Queensland Department of Education, covering organizational units, jobs, positions, locations, and their relationships within SAP SuccessFactors Employee Central and Organizational Management modules.

### 1. Core Organizational Data Objects

#### 1.1 Organizational Unit Objects (FR-1 to FR-5)
- **FOrganizationUnit** - Foundation object for organizational units
  - **Key Fields**: 
    - externalCode (Primary Key)
    - name, description
    - status (Active, Inactive, Planned)
    - startDate, endDate (date-effective)
    - parentId (hierarchical relationship)
    - costCenter, locationId (FK to respective objects)
    - headPositionId (FK to Position object)
    - transferRating
  - **Temporal Management**: Date-effective changes with effectiveStartDate/effectiveEndDate
  - **Relationships**: Hierarchical parent-child, linked to positions, locations, cost centers

- **OrganizationUnitMDF** - Master Data Framework extension object
  - **Custom Fields**: 
    - orgUnitType (Picklist: Department, Division, Region, School, Centre, Office, Branch, Business Unit)
    - establishmentControl (Funded, Over-Establishment)
    - fteCap (Maximum FTE allocation)
  - **Temporal Management**: Date-effective custom attributes
  - **Relationships**: Extension to FOrganizationUnit with additional attributes

#### 1.2 Job Objects (FR-9 to FR-13)
- **JobInfo** - Core job information object
  - **Key Fields**:
    - jobCode (Primary Key)
    - jobTitle, jobDescription
    - jobFamily, jobFunction
    - jobRoleLevel1, jobRoleLevel2
    - anzSCOCode, anzSCOName
    - certifiedAgreement
    - status (Active, Inactive, Planned)
    - startDate, endDate (date-effective)
  - **Temporal Management**: Date-effective job changes
  - **Relationships**: Linked to positions, job families, classifications

- **JobClassification** - Foundation object for job classifications
  - **Key Fields**:
    - externalCode (Primary Key)
    - name, description
    - jobFamilyId, jobFunctionId
    - level, grade
  - **Relationships**: Part of job architecture framework

#### 1.3 Position Objects (FR-14 to FR-19)
- **Position** - Core position object
  - **Key Fields**:
    - positionCode (Primary Key)
    - positionTitle, locallyKnownAs
    - jobCode (FK to JobInfo)
    - reportsTo, supervisorPosition
    - numberOfDirectReports
    - budgetedFTE, actualFTE
    - numberOfIncumbents, vacantFlag
    - positionType (Picklist: Permanent, Temporary, Casual, Contractor)
    - startDate, endDate (date-effective)
    - fundingType
    - departmentId, locationId, costCenterId (FK to respective objects)
    - award, classification
    - siteSpecificAllowance
    - hrDelegation, financeDelegation
    - identifiedPosition
  - **Temporal Management**: Date-effective position changes
  - **Relationships**: Links to jobs, organizational units, locations, cost centers, incumbents

- **PositionMDF** - Master Data Framework extension for positions
  - **Custom Fields**:
    - roleDescription (Attachment field for role descriptions)
    - concurrentEmploymentFlag
    - establishmentControl
  - **Relationships**: Extension to Position object with additional attributes

#### 1.4 Location Objects (FR-34 to FR-36)
- **Location** - Foundation object for locations
  - **Key Fields**:
    - externalCode (Primary Key)
    - name, address
    - locationContactDetails
    - latitude, longitude
    - status (Active, Inactive, Planned)
    - startDate, endDate (date-effective)
  - **Temporal Management**: Date-effective location changes
  - **Relationships**: Linked to organizational units, positions, cost centers

### 10. Data Validation and Business Rules

#### 10.1 Required Field Validation
- **Organization Unit**: externalCode, name, orgUnitType, status
- **Job**: jobCode, jobTitle, jobFamily, status
- **Position**: positionCode, positionTitle, jobCode, status
- **Location**: externalCode, name, address

#### 10.2 Format Validation
- **ID Fields**: Alphanumeric with specific length constraints
- **Date Fields**: ISO 8601 format
- **Decimal Fields**: FTE values with 2 decimal places maximum

#### 10.3 Cross-Reference Validation
- **Position to Job**: jobCode must exist in JobInfo
- **Position to Org Unit**: departmentId must exist in FOrganizationUnit
- **Position to Location**: locationId must exist in Location
- **Position to Cost Center**: costCenterId must exist in FCostCenter
