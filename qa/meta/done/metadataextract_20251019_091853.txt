# SAP SuccessFactors Metadata Extraction Report
## Concurrent Employment Optimization Configuration

### Executive Summary
This metadata extraction analyzes the concurrent employment optimization strategies document to identify core data objects, field definitions, relationships, and configuration parameters required for implementing advanced concurrent employment scenarios in SAP SuccessFactors Employee Central.

### 1. Core Data Objects Metadata

#### 1.1 Employee Central (EC) Objects
- **EmploymentInfo** - Employment relationship tracking
  - Key Fields: employmentStatus, startDate, endDate, employmentType
  - Temporal Management: Effective dating support for concurrent employment records
  - Relationships: One-to-Many with Person ID for multiple concurrent employments
  - Properties: Primary/Secondary employment indicators, FTE allocation fields

- **JobInfo** - Concurrent position assignment tracking
  - Key Fields: position, jobCode, employmentType, assignmentId, costCenter
  - Temporal Management: Date-effective concurrent position changes
  - Relationships: Many-to-One with Position object, One-to-Many with Person
  - Properties: Position hierarchy flags, reporting relationships

- **PersonalInfo** - Employee identity management across concurrent roles
  - Key Fields: personId, userId (multiple per person), employmentSwitcher
  - Relationships: One-to-Many mapping between Person ID and User IDs
  - Properties: Single sign-on context switching capabilities

#### 1.2 Position Management Objects
- **Position** - MDF Object for position hierarchy
  - Key Fields: positionCode, reportingToPosition, department, costCenter
  - Relationships: Hierarchical parent-child position relationships
  - Properties: Concurrent employment support flags, circular reference prevention

- **JobClassification** - MDF Object for role definitions
  - Key Fields: jobClassificationCode, jobFamily, jobFunction
  - Relationships: Position-to-JobClassification associations
  - Properties: Concurrent employment eligibility indicators
