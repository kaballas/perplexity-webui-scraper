# SAP SuccessFactors Metadata Extraction Report
## Queensland Department of Education HCMS - Manage Onboarding Requirements Analysis

### Executive Summary
This metadata extraction analyzes the technical and data model requirements for the "Manage Onboarding" functional area (FR-39 to FR-51) within the Queensland Department of Education's HCMS implementation. The analysis covers Employee Central (EC), Onboarding 2.0, and integration metadata necessary to support the specified functional requirements.

---

## 1. Core Data Objects Metadata

### 1.1 Employee Central (EC) Objects
- **PersonalInfo** - Personal information capture and storage
  - Key Fields: firstName, lastName, gender, dateOfBirth, email, phone, address
  - Temporal Management: Effective dating for personal information changes
  - Required for: FR-39 (Name and contact information)

- **EmploymentInfo** - Employment history and status tracking
  - Key Fields: employmentStatus, startDate, endDate, jobInfo changes
  - Temporal Management: Effective dating support for historical employment records
  - Required for: FR-41, FR-42, FR-43 (Appointment and employment details)

- **JobInfo** - Job assignment and position history
  - Key Fields: position, jobCode, employmentType, assignmentId
  - Temporal Management: Date-effective job changes
  - Required for: FR-41, FR-43 (Position details, concurrent employment)

- **PayComponentNonRecurring** - Banking and deduction details
  - Key Fields: bankAccount, deductionCodes, paymentMethod
  - Temporal Management: Effective dating for payment changes
  - Required for: FR-39 (Banking details, employee deductions)

- **PerPerson** - Core person data with emergency contact support
  - Key Fields: emergencyContact, personalEmail, homePhone
  - Temporal Management: Effective dating for personal data
  - Required for: FR-39 (Emergency contact details)

### 1.2 Onboarding 2.0 MDF Objects
- **OnboardingProcess** - Onboarding workflow and status tracking
  - Key Fields: processId, status, startDate, onboardingSteps
  - Temporal Management: Process start and completion dates
  - Required for: FR-44, FR-45 (Early onboarding support)

- **OnboardingStep** - Individual onboarding task definitions
  - Key Fields: stepName, stepStatus, assignee, completionDate
  - Temporal Management: Step-level effective dating
  - Required for: FR-46, FR-47 (Training and induction distribution)

### 1.3 Custom MDF Objects for Superannuation
- **SuperannuationDetails** - Superannuation fund information
  - Key Fields: fundName, fundNumber, memberNumber, employerChoice
  - Temporal Management: Effective dating from start of employment
  - Required for: FR-39 (Superannuation details)

## 7. Technical Considerations and Potential Issues

### 7.1 Integration Complexity Issues
- **Issue**: Real-time data synchronization between Onboarding 2.0 and Employee Central
- **Metadata Impact**: Requires OData API monitoring and error handling
- **Required For**: FR-40, FR-42, FR-43 (Automatic transfer requirements)

### 7.2 Concurrent Employment Data Model Issues
- **Issue**: Managing multiple assignments and funding streams
- **Metadata Impact**: Complex Position-to-Job relationships with effective dating
- **Required For**: FR-43 (Concurrent employment support)

### 7.3 Early Onboarding Process Issues
- **Issue**: Managing pre-qualified applicants with future start dates
- **Metadata Impact**: Employment status management with future effective dates
- **Required For**: FR-44, FR-45 (Early onboarding requirements)

### 7.4 No-Show Process Issues
- **Issue**: End-dating employee records and payroll recovery
- **Metadata Impact**: EmploymentInfo status management and audit trails
- **Required For**: FR-50 (No-show management)
