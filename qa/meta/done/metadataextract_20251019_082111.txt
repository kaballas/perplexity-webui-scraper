# SAP SuccessFactors Work Schedule Finder Metadata Extraction Report

## Executive Summary
This report provides comprehensive metadata extraction for the Work Schedule Finder enhancement project within the Queensland Department of Education's HCMS Programme. The analysis covers current state limitations, future state requirements, and technical specifications for the BTP extension.

## 2. Core Object Definitions

### 2.1 Work Schedule Models
The system supports three distinct work schedule models:

#### 2.1.1 Simple Model
- **Object Type**: Standard Schedule Configuration
- **Properties**: Basic weekly pattern definition
- **Use Case**: Standard full-time/part-time schedules
- **Data Structure**: Day-based hour allocation

#### 2.1.2 Period Model  
- **Object Type**: Period-based Schedule Configuration
- **Properties**: Multi-period scheduling capability
- **Use Case**: Extended work patterns beyond weekly cycles
- **Data Structure**: Period-specific hour allocation

#### 2.1.3 Schedule Model
- **Object Type**: Advanced Schedule Configuration
- **Properties**: Complex multi-week patterns
- **Use Case**: Rotating shifts, complex rosters
- **Data Structure**: Multi-dimensional schedule matrix

### 2.2 Work Schedule Finder Application
- **Component Type**: BTP Extension Application
- **Integration Point**: EC People Profile (Job Information section)
- **Functionality**: Enhanced search and assignment capability
- **Access Level**: Manager and Admin permissions

## 3. Field Definitions and Properties

### 3.1 Search Criteria Fields

#### 3.1.1 Hours per Week
- **Field Name**: `hoursPerWeek`
- **Data Type**: Decimal (Numeric)
- **Precision**: 2 decimal places
- **Range**: 0.00 to 168.00 (max hours in week)
- **Validation**: Positive values only
- **Required**: Optional
- **Filter Type**: Range filter

#### 3.1.2 Hours per Pay Period
- **Field Name**: `hoursPerPayPeriod`
- **Data Type**: Decimal (Numeric)
- **Precision**: 2 decimal places
- **Validation**: Positive values only
- **Required**: Optional
- **Filter Type**: Range filter

#### 3.1.3 Day by Day Inputs
- **Field Name**: `dayByDaySchedule`
- **Data Type**: Complex Object Array
- **Structure**: 
  - `dayOfWeek`: String (MON, TUE, WED, THU, FRI, SAT, SUN)
  - `hours`: Decimal (0.00 to 24.00)
  - `startTime`: Time Format
  - `endTime`: Time Format
- **Validation**: Non-overlapping time periods
- **Required**: Conditional

#### 3.1.4 Multi-Week Roster
- **Field Name**: `multiWeekRoster`
- **Data Type**: Boolean
- **Values**: TRUE/FALSE
- **Default**: FALSE
- **Required**: Optional
- **Filter Type**: Checkbox filter

#### 3.1.5 Worker Type
- **Field Name**: `workerType`
- **Data Type**: String (Picklist Reference)
- **Values**: 
  - `FULL_TIME`
  - `PART_TIME`
  - `CASUAL`
  - `CONTRACT`
  - `TEMPORARY`
- **Required**: Optional
- **Filter Type**: Multi-select dropdown

#### 3.1.6 Schedule Type
- **Field Name**: `scheduleType`
- **Data Type**: String (Picklist Reference)
- **Values**:
  - `SIMPLE`
  - `PERIOD`
  - `SCHEDULE`
- **Required**: Optional
- **Filter Type**: Single-select dropdown

### 3.2 Additional Functionality Fields

#### 3.2.1 Schedule Management Fields
- **Copy Function**: `copyScheduleEnabled` (Boolean)
- **Create Function**: `createScheduleEnabled` (Boolean)
- **Real-time Creation**: `realTimeCreation` (Boolean)
- **Replication Status**: `plannedWorkingTimeReplicated` (Boolean)
