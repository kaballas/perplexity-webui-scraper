# SAP SuccessFactors Organization Design Metadata Extraction Report

## Executive Summary
This metadata extraction report analyzes the organizational design requirements for the Queensland Department of Education HCMS implementation, focusing on the metadata structures required to support the 38 functional requirements for organization design management.

## 1. Core Foundation Objects Metadata

### 1.1 Organizational Units (FODepartment)
- **Object Name**: FODepartment
- **Purpose**: Support organizational units including department, divisions, regions, schools/centres, offices, branches, business units
- **Key Fields**:
  - `externalCode` - Org Unit ID
  - `name` - Org Unit Title
  - `status` - Status (Active/Inactive)
  - `headOfUnit` - Head Position ID
  - `costCenter` - Cost Centre reference
  - `location` - Location reference
  - `transferRating` - Transfer Rating
  - `startDate`/`endDate` - Date effective management
- **Temporal Management**: Effective dating support for historical organizational changes
- **Required for**: FR-1, FR-2, FR-3, FR-5

### 1.2 Job Foundation Object (FOJobCode)
- **Object Name**: FOJobCode
- **Purpose**: Support job management with date-effective capabilities
- **Key Fields**:
  - `externalCode` - Job ID
  - `jobFamily` - Job Family reference
  - `jobFunction` - Job Function
  - `jobRoleL1` - Job Role Level 1
  - `jobRoleL2` - Job Role Level 2
  - `anzscoCode` - ANZSCO Code
  - `anzscoName` - ANZSCO Name
  - `certifiedAgreement` - Certified Agreement
  - `startDate`/`endDate` - Date effective management
- **Temporal Management**: Effective dating support for job changes
- **Required for**: FR-9, FR-10, FR-11, FR-12, FR-13

### 1.3 Cost Center Foundation Object (FOCostCenter)
- **Object Name**: FOCostCenter
- **Purpose**: Support cost center management
- **Key Fields**:
  - `externalCode` - Cost Center ID
  - `name` - Cost Center Name
  - `startDate`/`endDate` - Date effective management
- **Required for**: FR-1, FR-2, FR-15

### 1.4 Location Foundation Object (FOLocation)
- **Object Name**: FOLocation
- **Purpose**: Support location/centre management
- **Key Fields**:
  - `externalCode` - Location ID
  - `name` - Location Name
  - `address` - Location Address
  - `contactDetails` - Location Contact Details
  - `latitude`/`longitude` - Geographic coordinates
  - `startDate`/`endDate` - Date effective management
- **Temporal Management**: Effective dating support for location changes
- **Required for**: FR-1, FR-34, FR-35, FR-36

## 10. Metadata Summary and Dependencies

The metadata extraction reveals that the organizational design requirements will need:

1. **Foundation Objects**: Enhanced FODepartment, FOJobCode, FOCostCenter, FOLocation
2. **MDF Objects**: Custom Position object with extensive attributes
3. **Integration Objects**: JobInfo and EmploymentInfo for concurrent employment
4. **Custom Picklists**: Position type, funding type, and status picklists
5. **Business Rules**: FTE validation and approval workflows
6. **Reporting Objects**: Comprehensive organizational reporting cubes

This metadata foundation supports all 38 functional requirements for organization design management while maintaining date-effective processing and hierarchical organizational structure management capabilities.
