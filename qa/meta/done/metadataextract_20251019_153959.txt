## 5. Workforce Schema API Endpoints

### 5.1 OData API Endpoints for Candidate Data

**GET /odata/v2/Candidate**
```
- Filters supported: personId, email, firstName, lastName
- Expands supported: education, employmentHistory, applications
- Select fields: personId, firstName, lastName, email, phone, dateOfBirth
- Rate limit: 100 requests/minute
- Authentication: OAuth2 Bearer token
```

**GET /odata/v2/Application**
```
- Filters supported: applicationId, candidateId, postingId, status
- Expands supported: candidate, questionnaire, attachments
- Select fields: applicationId, candidateId, postingId, applicationDate, status
- Rate limit: 100 requests/minute
- Authentication: OAuth2 Bearer token
```

**GET /odata/v2/AICandidateSummary**
```
- Filters supported: applicationId, summaryStatus, generatedDate
- Expands supported: application, candidate
- Select fields: summaryId, applicationId, generatedDate, summaryStatus, confidenceScore
- Rate limit: 50 requests/minute (custom MDF object)
- Authentication: OAuth2 Bearer token
```

### 5.2 Custom MDF API Endpoints

**GET /odata/v2/AICandidateSummary**
```
- EntitySet: AICandidateSummary
- Key: summaryId
- Supported operations: Read, Create, Update
- Custom validation: Requires human validation for confidenceScore < 0.7
- Business rules: Auto-approval for confidenceScore >= 0.85
```

**GET /odata/v2/AIProcessingConfiguration**
```
- EntitySet: AIProcessingConfiguration
- Key: configId
- Supported operations: Read, Create, Update, Delete
- Effective dating: Yes (effectiveStartDate, effectiveEndDate)
- Validation: Only one active configuration per configType
```

## 8. Workflow and Business Rule Metadata

### 8.1 AI Processing Workflow

**Workflow: GenerateCandidateSummary**
```
- Trigger: New Application Status = "REVIEW"
- Steps:
  1. Validate required documents exist
  2. Extract candidate data via OData API
  3. Process documents via AI engine
  4. Generate summary using configured template
  5. Calculate confidence score
  6. Apply compliance checks
  7. Create AICandidateSummary record
  8. Route based on confidence score and risk indicators
- Conditions: 
  - If confidenceScore >= 0.85: Auto-approve
  - If confidenceScore < 0.7: Human validation required
  - If riskIndicators detected: Compliance review required
```

### 8.2 Business Rules Configuration

**Rule: Compliance Check**
```
- Condition: Any compliance requirement not met
- Action: Set summaryStatus = "VALIDATION_REQUIRED"
- Notification: Send to Compliance Officer
- Log: Record in compliance audit trail
```

**Rule: Bias Detection**
```
- Condition: Demographic patterns detected in scoring
- Action: Set biasDetected = True, riskLevel = "HIGH"
- Notification: Alert Compliance Officer
- Log: Bias detection event in audit trail
```
