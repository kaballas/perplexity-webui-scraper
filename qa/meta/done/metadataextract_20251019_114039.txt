# SAP SuccessFactors Metadata Extraction Report
## Queensland Department of Education - Recruitment Management System

### Executive Summary
This metadata extraction focuses on the recruitment and hiring processes for the Queensland Department of Education, covering both corporate/region and teacher/school-based hiring scenarios within SAP SuccessFactors Recruiting Management (RCM).

### 1. Core Recruitment Data Objects

#### 1.1 Requisition Management Objects
- **JobRequisition** - Core requisition entity
  - **Key Fields**: 
    - requisitionId (Primary Key)
    - title, description
    - jobCode, positionId (FK to Position MDF)
    - status (Active, Closed, On Hold, etc.)
    - templateType (Corporate/Teacher/EOI/Exception)
    - creationMethod (Position Management/Copy-Created)
    - establishmentControl (Funded/Over-Establishment)
  - **Temporal Management**: Date-effective status changes
  - **Relationships**: Links to Position MDF object, Job Classification FO

- **Position** (MDF Object) - Position Management Integration
  - **Key Fields**: 
    - positionId (Primary Key)
    - positionTitle, jobCode (FK)
    - departmentCode (FK to FODepartment)
    - costCenter (FK to FOCostCenter)
    - FTEValue, employmentType
    - positionStatus (Active/Vacant/Filled)
  - **Temporal Management**: Effective dating for position changes
  - **Dependencies**: Employee Central EmploymentInfo

#### 1.2 Candidate Management Objects
- **Candidate** - Core candidate profile
  - **Key Fields**:
    - candidateId (Primary Key)
    - personalInfo (firstName, lastName, email, phone)
    - workingRightsStatus
    - conflictOfInterestDeclaration
    - jobReadyStatus (for Evergreen requisitions)
    - talentPoolMembership
  - **Relationships**: Multiple applications, requisition assignments

- **Application** - Candidate application tracking
  - **Key Fields**:
    - applicationId (Primary Key)
    - candidateId (FK), requisitionId (FK)
    - applicationStatus (New, Screening, Interview, Offer, Hired)
    - sourceChannel (External Career Site, LinkedIn, SmartJobs, Internal)
    - screeningResults, eligibilityStatus
  - **Temporal Management**: Status change history with timestamps

### 2. Requisition Template Metadata

#### 2.1 Corporate/Region Job Requisition Template
- **Template Code**: CORP_JOB_REQ
- **Required Fields**: 
  - positionId, jobCode, department, costCenter
  - hiringManagerId, uplineManagerId
  - recruitmentTeamId
  - classificationDetails
- **Workflow Configuration**: 
  - Approval Steps: Hiring Manager → Upline Manager → Corporate Recruitment
  - Conditional Logic: Establishment control validation
- **Business Rules**: One-to-one position-to-requisition relationship enforcement

#### 2.2 Teacher Job Requisition Template
- **Template Code**: TEACHER_JOB_REQ
- **Required Fields**:
  - positionId, jobCode, schoolId
  - subjectArea, teachingLevel
  - FTEValue, contractType
  - principalId, regionalHRId
- **Workflow Configuration**:
  - Approval Steps: Principal → Regional HR → Endorser → School Delegate
- **Special Logic**: Multiple FTE split handling (separate requisitions per position)

#### 2.3 Evergreen Requisition Templates
- **Teacher Eligibility Job Requisition (Evergreen)**
  - **Template Code**: TEACHER_ELIGIBILITY_EVG
  - **Key Characteristics**: 
    - No specific positionId linkage
    - Continuous intake capability
    - Talent pool management
    - Job Ready status tracking
  - **Candidate Status Fields**: New Application, Identity Verification, Future Teachers, Job Ready

- **Generic Job Requisition (Evergreen)**
  - **Template Code**: GENERIC_EVG
  - **Purpose**: General talent acquisition for future opportunities

#### 2.4 Specialized Requisition Templates
- **Exception from Advertising Requisition**
  - **Template Code**: EXC_FROM_ADV
  - **Use Cases**: Compassionate transfers, medical return to work, closed merit processes
  - **Required Fields**: Exception reason codes, policy references
  - **Approval Workflow**: Policy-based approval routing

- **Expression of Interest (EOI) Requisition**
  - **Template Code**: EOI_REQ
  - **Use Cases**: Temporary/internal opportunities
  - **Workflow**: Internal approval process only

- **School Based Classified Officer Job Requisition**
  - **Template Code**: CLASSIFIED_OFFICER_REQ
  - **Target**: Permanent DoE teachers for relocation

- **School Support Job Requisition**
  - **Template Code**: SCHOOL_SUPPORT_REQ
  - **Target**: Non-teaching administrative/support roles

### 3. Workflow and Approval Metadata

#### 3.1 Approval Workflow Definitions
- **Object**: ApprovalRouteMap
  - **Key Fields**:
    - workflowId, templateType
    - approvalSequence, approverType
    - conditionalRules, escalationPaths
  - **Approver Types**: Role-based, Dynamic Group, Dynamic Role
  - **Derivation Source**: Workflow Derivation Lookup Table

#### 3.2 Workflow Participant Roles
- **Approvers**: Primary decision makers with approval authority
- **Contributors**: Provide input/feedback during process
- **CC Roles**: Receive notifications and updates
- **Dynamic Routing**: Based on Position attributes, Cost Center, or Department

#### 3.3 Delegation Matrix Integration
- **Delegation Rules**: Support for manager delegation and proxy approvals
- **Authority Levels**: Configured based on role and organizational hierarchy
- **Escalation Procedures**: Automated escalation for pending approvals

### 4. Picklist Metadata

#### 4.1 Requisition Status Picklists
- **REQ_STATUS**: Active, In Approval, Approved, Rejected, On Hold, Closed, Cancelled
- **APP_STATUS**: New, Screening, Interview, Assessment, Offer, Hired, Rejected
- **POSITION_STATUS**: Active, Vacant, Filled, Inactive, Frozen

#### 4.2 Recruitment Source Picklists
- **SOURCE_CHANNEL**: External Career Site, LinkedIn, SmartJobs, Internal Postings, Referral, Agency
- **REQUISITION_TEMPLATE**: All template codes defined above
- **CONTRACT_TYPE**: Permanent, Temporary, Fixed Term, Casual
- **EMPLOYMENT_TYPE**: Full-time, Part-time, Casual

### 5. Business Rules and Validation Metadata

#### 5.1 Establishment Control Rules
- **Rule**: One-to-one position-requisition relationship enforcement
- **Validation**: Prevents multiple active requisitions for same positionId
- **Exception**: Over-establishment requires ServiceNow approval

#### 5.2 FTE Split Management
- **Rule**: Multiple FTE splits require separate requisitions
- **Validation**: Each 0.5 FTE requires distinct positionId
- **Process**: System-enforced separation for accurate tracking

#### 5.3 Talent Pool Management
- **Rule**: Evergreen requisitions remain open indefinitely
- **Process**: Candidate movement from eligibility to specific requisitions
- **Status Management**: Segregation through candidate status field

### 6. Integration Metadata

#### 6.2 External System Integration
