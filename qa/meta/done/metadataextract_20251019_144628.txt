# SAP SuccessFactors Internal Secondment Metadata Extraction Report

## Executive Summary
This metadata extraction analyzes the data model requirements for processing date-effective internal secondments within SAP SuccessFactors Employee Central, focusing on the automatic defaulting of employment details from secondment positions with HR/Payroll override capabilities and automatic end action creation.

## 1. Core Data Objects for Internal Secondment Processing

### 1.1 Employee Central (EC) Objects
- **EmploymentInfo** - Employment history and status tracking
  - Key Fields: employmentStatus, startDate, endDate, jobInfo changes
  - Temporal Management: Effective dating support for historical employment records
  - Required for: Internal secondment start/end date management

- **JobInfo** - Job assignment and position history
  - Key Fields: position, jobCode, employmentType, assignmentId
  - Temporal Management: Date-effective job changes for secondment assignments
  - Required for: Position-based employment detail defaulting

- **PersonalInfo** - Employee personal data
  - Key Fields: employeeNumber, firstName, lastName, email
  - Temporal Management: Historical personal information tracking

### 1.2 Master Data Framework (MDF) Objects
- **Position** - Position management object
  - Key Fields: positionId, positionTitle, jobCode, costCenter, department
  - Temporal Management: Position changes effective dating
  - Required for: Secondment position details and employment detail defaulting

- **Custom Secondment Object** - Internal secondment specific data
  - Key Fields: secondmentId, secondmentType, secondmentStartDate, secondmentEndDate
  - Temporal Management: Date-effective secondment processing
  - Required for: Tracking secondment-specific attributes

## 2. Provisioning Enablement Metadata

### 2.1 Company Settings Configuration
- **Feature**: Higher Duty or Temporary Assignment Enablement
- **Location**: Provisioning > Company Settings > Industry Features
- **Status**: Enabled/Disabled flag
- **Dependencies**: Employee Central and Position Management modules

### 2.2 Concurrent Employment Configuration
- **Feature**: Concurrent Employment Support
- **Location**: Provisioning > Company Settings > Employee Central
- **Status**: Required for internal secondments
- **Dependencies**: EmploymentInfo object with multiple active records

## 3. Permission Configuration Metadata

### 3.1 Role-Based Permissions Structure
- **Permission Category**: Higher Duty or Temporary Assignment
- **Permission Types**:
  - Add Higher Duty or Temporary Assignment
  - End Higher Duty or Temporary Assignment  
  - View Higher Duty or Temporary Assignment details

### 3.2 Permission Role Requirements
- **HR Officers**: Full secondment creation and modification permissions
- **Payroll Officers**: Override capabilities for employment details
- **Managers**: View and limited modification permissions
- **Self-Service**: Employee viewing of secondment details

### 3.3 Field-Level Security Configuration
- **Restricted Fields**: Pay components, cost center allocations
- **Editable by HR**: Employment details, position assignments
- **Editable by Payroll**: Compensation overrides, pay grade adjustments

## 4. Higher Duty/TA Configuration Settings Metadata

### 4.1 Core Configuration Parameters
- **Automatic Termination**: Flag for job-based termination processing
- **Base Pay Component Group**: Pay components for base pay calculations
- **Annualized Pay Component Group**: Components including base pay and allowances
- **Applicable HD/TA Pay Component Group**: Subset for HD allowance calculations
- **Estimated HD Allowance Component**: Recurring pay component for allowance display

### 4.2 Event Reason Configuration
- **Event Reason for Comp Info Updates**: Code for automatic pay component creation
- **Allow Permission Bypass**: Flag for permission model flexibility
- **Values**: "Yes" (HD-specific permissions only) / "No" (standard employment permissions required)

## 5. Workflow Configuration Metadata

### 5.1 Workflow Definitions
- **HRIS Action**: Higher Duty/Temporary Assignment workflows
- **Approval Levels**: Multi-level approval based on organizational hierarchy
- **Routing Rules**: Position-based and responsibility percentage routing

### 5.2 Event Reason Picklist Configuration
- **Location**: Corporate Data Model > `<eventReason>` HRIS element
- **Picklist ID**: Must match HRIS field `<event>` of `<eventReason>` element
- **External Codes**: Must remain unchanged for system compatibility

## 6. Business Rules Configuration Metadata

### 6.1 Position-Based Defaulting Rules
- **Rule Type**: Employment detail defaulting from secondment position
- **Defaulted Fields**:
  - Organizational Unit (from position)
  - Job Code (from position)
  - Pay Grade (from position)
  - Cost Center (from position)
  - Work Location (from position)
  - Manager (from position)

### 6.2 Validation Rules
- **Mandatory Fields**: Secondment start/end dates
- **Date Validation**: End date must be after start date
- **Position Validation**: Valid position assignment required
- **Responsibility Percentage**: 100% for full responsibility scenarios

### 6.3 Automatic End Action Creation Rules
- **Trigger Condition**: Secondment start action creation
- **End Action Type**: Return to substantive position
- **Defaulting**: Original employment details restoration
- **Right to Return**: Applies only to 100% Full Responsibility assignments
