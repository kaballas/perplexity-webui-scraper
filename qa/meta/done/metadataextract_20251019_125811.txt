# SAP SuccessFactors Metadata Extraction Report: Teacher Transfer Points System

## Executive Summary
This metadata extraction focuses on the Queensland Department of Education's Teacher Transfer Points system, analyzing the custom BTP (Business Technology Platform) application integrated with SuccessFactors Employee Central. The analysis covers the comprehensive data architecture required for transfer point calculations, eligibility management, and approval workflows that exceed standard RCM capabilities.

## Core Foundation Object Metadata

## Business Rule Metadata

### TransferPointAccrualRule
```
Rule Name: TransferPointAccrualRule
Purpose: Calculate daily transfer point accrual
Trigger: JobInfo update (startDate, endDate, company, department)
Logic:
  IF JobInfo.endDate IS NULL OR JobInfo.endDate >= TODAY
    THEN calculate daily accrual based on current rating
  ELSE calculate accrual for completed period only
Calculation:
  dailyAccrual = (ratingLevel / totalSchoolDays) * partTimeFactor
  totalPoints = SUM(dailyAccrual for each service day)
Validation: Points cannot exceed maximum for rating level
Security: System-generated, no user access
```

### ServiceContinuityRule
```
Rule Name: ServiceContinuityRule
Purpose: Validate continuous service requirements
Trigger: JobInfo end date change
Logic:
  IF breakInService > 60 school days
    THEN reset continuity counter
    ELSE maintain continuous service
Exception: Unpaid leave < 30 days maintains continuity
Validation: Check for breaks exceeding 60 school days
Security: System-generated, no user access
```

### TransferEligibilityRule
```
Rule Name: TransferEligibilityRule
Purpose: Validate transfer eligibility requirements
Trigger: TransferApplication submission
Logic:
  IF serviceDays < 60 school days
    THEN reject with insufficient service
  ELSE IF currentPoints < minimumRequired
    THEN reject with insufficient points
  ELSE approve eligibility
Validation: Minimum service and points requirements
Security: Applied during application submission
```
