# SAP SuccessFactors Metadata Extraction Report
## Auto-Increment and Pay Scale Progression Enhancement Analysis

### Executive Summary
This metadata extraction analyzes the Employee Central Pay Scale Progression functionality and identifies required customizations for Queensland Department of Education's public sector requirements, specifically addressing Leave Without Pay (LWOP) suspension, Higher Duties service aggregation, and complex award rule compliance.

## 1. Core Standard Objects and Fields

### 1.1 Pay Scale Progression Standard Objects
- **Object Name**: `payScaleLevel` (MDF Object)
- **Purpose**: Defines pay scale levels and progression criteria
- **Key Fields**:
  - `payScaleLevelEntryDate` (Date type) - Entry date for current pay scale level
  - `membershipDurationForProgression` (Integer) - Required months/days for progression
  - `payScaleLevelCode` (String) - Unique identifier for pay scale level
  - `payScaleCode` (String) - Parent pay scale structure reference

### 1.2 Job Information Standard Fields
- **Object Name**: `jobInfo`
- **Key Progression Fields**:
  - `payScaleLevel` (Picklist reference) - Current pay scale level
  - `payScaleCode` (Picklist reference) - Associated pay scale structure
  - `processAutoPayGradeChange` (Boolean) - Manual override flag for progression
  - `startDate` (Date) - Job assignment start date
  - `payGrade` (Picklist) - Associated pay grade structure

### 1.3 Off Cycle Event Batch Components
- **Object Name**: `offCycleEvent` (MDF Object)
- **Purpose**: Batch processing for auto-increment events
- **Filter Fields**:
  - `employeeFilter` - Selection criteria for eligible employees
  - `eventDate` - Processing date for eligibility calculations
  - `businessRuleReference` - Associated business rule for processing

## 2. Custom MDF Object Definitions

### 2.1 SubstantiveServiceTracker (Proposed)
- **Object Name**: `SubstantiveServiceTracker` (Custom MDF)
- **Purpose**: Track accumulated Higher Duties service for substantive position progression
- **Field Definitions**:
  - `employeeId` (String, Required) - Employee identifier (Association to User)
  - `substantivePositionId` (String) - Substantive position reference
  - `substantiveEntryDate` (Date) - Original substantive pay scale level entry date
  - `accumulatedHDServiceDays` (Integer) - Total HD days counting toward substantive progression
  - `adjustedIncrementEligibilityDate` (Date) - Recalculated eligibility date with HD service
  - `lastHDAssignmentEndDate` (Date) - Date of most recent HD assignment end
  - `hdServicePeriods` (String) - Comma-separated list of HD service periods

### 2.2 LWOPAdjustmentTracker (Proposed)
- **Object Name**: `LWOPAdjustmentTracker` (Custom MDF)
- **Purpose**: Track LWOP periods affecting pay scale progression
- **Field Definitions**:
  - `employeeId` (String, Required) - Employee identifier
  - `lwopPeriodStart` (Date) - LWOP period start date
  - `lwopPeriodEnd` (Date) - LWOP period end date
  - `lwopDaysAccumulated` (Integer) - Total LWOP days affecting progression
  - `adjustedEntryDate` (Date) - Progression entry date with LWOP exclusions
  - `originalEntryDate` (Date) - Original pay scale level entry date
  - `lwopType` (Picklist) - Type of LWOP (Personal, Medical, Career, etc.)

## 3. Custom Field Additions to Standard Objects

### 3.1 Job Information Custom Fields
- **Field Name**: `totalLWOPDaysSinceEntry`
  - **Type**: Integer
  - **Purpose**: Track total LWOP days since pay scale level entry
  - **Default Value**: 0
  - **Visibility**: HR Admin only

- **Field Name**: `adjustedPayScaleLevelEntryDate`
  - **Type**: Date
  - **Purpose**: Entry date adjusted for LWOP exclusions
  - **Calculation**: Business Rule driven
  - **Visibility**: HR Admin only

- **Field Name**: `hdServiceAccumulatedForSubstantive`
  - **Type**: Integer
  - **Purpose**: HD service days applicable to substantive position
  - **Default Value**: 0
  - **Visibility**: HR Admin only

### 3.2 Pay Scale Level Custom Fields
- **Field Name**: `awardSpecificIncrementRules`
  - **Type**: String (Text Area)
  - **Purpose**: Store award-specific progression rules
  - **Visibility**: System configuration only

- **Field Name**: `performanceGatingRequired`
  - **Type**: Boolean
  - **Purpose**: Flag for performance-based progression requirements
  - **Default Value**: False

## 4. Business Rules Metadata

### 4.1 Standard Pay Scale Progression Rule
- **Rule Name**: `AutoIncrementEligibilityCheck`
- **Object Context**: `jobInfo`
- **Trigger**: onSave, onChange
- **Logic Type**: IF-THEN eligibility validation
- **Standard Condition**: 
  ```
  IF (TODAY - payScaleLevelEntryDate) >= (membershipDurationForProgression * 30)
  THEN eligibleForProgression = TRUE
  ```

### 4.2 Enhanced LWOP-Aware Rule (Proposed)
- **Rule Name**: `LWOPAdjustedAutoIncrementCheck`
- **Object Context**: `jobInfo`
- **Trigger**: onSave, onChange, Integration Center events
- **Enhanced Logic**:
  ```
  adjustedServiceDays = (TODAY - payScaleLevelEntryDate) - totalLWOPDaysSinceEntry
  IF adjustedServiceDays >= (membershipDurationForProgression * 30)
  THEN eligibleForProgression = TRUE
  ELSE eligibleForProgression = FALSE
  ```

### 4.3 Higher Duties Service Aggregation Rule (Proposed)
- **Rule Name**: `HDServiceAggregationRule`
- **Object Context**: `SubstantiveServiceTracker`
- **Trigger**: HD assignment end, jobInfo changes
- **Logic**:
  ```
  totalService = (TODAY - substantiveEntryDate) + accumulatedHDServiceDays
  IF totalService >= eligibilityThreshold
  THEN incrementSubstantivePosition = TRUE
  ```

- **Job Name**: `LWOPServiceTrackerIntegration`
- **Schedule**: Daily (Nightly)
- **Source**: Time Off API - Absence records
- **Target**: `LWOPAdjustmentTracker` MDF object
- **Filter Criteria**: 
  - Absence Type: LWOP categories
  - Status: Approved
  - Period: Current or recent

### 5.2 Higher Duties Integration Job
- **Job Name**: `HDServiceIntegration`
- **Schedule**: Event-driven (HD assignment end)
- **Source**: Job Information API
- **Target**: `SubstantiveServiceTracker` MDF object
- **Trigger**: Job assignment end with HD indicators

## 7. Workflow and Process Metadata

### 7.1 LWOP Auto-Flag Workflow
- **Workflow Name**: `LWOPProgressionFlagManagement`
- **Trigger**: Time Off approval for LWOP types
- **Actions**:
  - Set `processAutoPayGradeChange = FALSE`
  - Create `LWOPAdjustmentTracker` record
  - Notify HR Admin of progression suspension

### 7.2 HD Service Credit Workflow
- **Workflow Name**: `HDCreditServiceWorkflow`
- **Trigger**: HD assignment end/return to substantive
- **Actions**:
  - Calculate HD service duration
  - Update `SubstantiveServiceTracker`
  - Recalculate substantive progression eligibility
